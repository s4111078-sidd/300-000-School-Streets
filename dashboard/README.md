# 300,000 Streets — School Safety Dashboard

Static web dashboard for the Regen Melbourne school streets safety analysis.
Built with Leaflet.js, Chart.js, and Tailwind CSS — no build step required.

---

## How to run

### 1. Run the main analysis pipeline

```bash
python run_all.py
```

This produces all CSV outputs and chart images in `outputs/`.

### 2. Generate the dashboard data

```bash
python prepare_data.py
```

Reads `outputs/hs_scores.csv`, `outputs/recommendations.csv`,
`outputs/ml_predictions.csv`, `outputs/seifa_darebin.csv`, and `config.py`,
then writes:

- `dashboard/data/data.json` — all school data in one JSON file
- `dashboard/data/charts/` — copies of all six chart PNGs

Re-run this script any time the pipeline outputs change to update the dashboard.

### 3. Open the dashboard locally

Open `dashboard/index.html` in your browser.

> **Note:** Because the dashboard fetches `data.json` via `fetch()`, it must be
> served from a local HTTP server, not opened as a `file://` URL directly.
>
> Quickest option (Python):
> ```bash
> python -m http.server 8000 --directory dashboard
> ```
> Then open http://localhost:8000 in your browser.
>
> Alternatively use the VS Code "Live Server" extension.

## GitHub Pages deployment

1. Push the `frontend-dashboard` branch to GitHub.
2. Go to your repository on GitHub.
3. Navigate to **Settings → Pages**.
4. Under **Source**, choose **Deploy from a branch**.
5. Set **Branch** to `frontend-dashboard` and **Folder** to `/dashboard`.
6. Click **Save**.

The dashboard will be live in about 2 minutes at:

```
https://s4111078-sidd.github.io/300-000-School-Streets/
```

## Updating the dashboard

The dashboard reads **only** from `dashboard/data/data.json`. To update it:

1. Edit `school_data.csv` with new field observations.
2. Run `python run_all.py` to regenerate pipeline outputs.
3. Run `python prepare_data.py` to refresh `data.json`.
4. Reload the browser.

No manual data entry in the dashboard files is ever needed.
